# Avivo.ai Internal Policy Guidelines

## Communication
- Clear and respectful communication.
- Document project updates.
- Escalate blockers early.

## Code of Conduct
- Write modular, readable code.
- Avoid unnecessary dependencies.
- Ensure reproducibility with environment files.

## Deployment Rules
- Reviewed by at least one engineer.
- Restricted production access.
- Logs must not contain sensitive data.

## Security Standards
- Use environment variables for secrets.
- Rotate keys every 90 days.
